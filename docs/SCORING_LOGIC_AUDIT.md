## 1. 시스템 개요

### 데이터 파이프라인

```
Yahoo Finance 비공식 API
  ├─ 스크리너 API → 종목 심볼 목록 수집 (최대 20개)
  ├─ 차트 API → 개별 종목 상세 데이터 추출
  └─ 45초 인메모리 캐시
        ↓
  스코어링 엔진 (12기준 × 2점 = 최대 24점)
        ↓
  시그널 판정 (HIGH ≥18 / MODERATE ≥10 / LOW <10)
        ↓
  진입 전략 생성 (고정 % 기반 손절/목표가)
```

### 데이터 흐름 상세

1. **종목 수집**: Yahoo Finance 스크리너/트렌딩 API에서 심볼 목록 수집
2. **데이터 추출**: 각 심볼의 1분봉 차트에서 가격/거래량/52주 고저 추출
3. **거래량 비율 산출**: `프리마켓 총 거래량 / (평균일거래량 × 0.05)` — 프리마켓이 하루의 5%라는 가정
4. **촉매 자동 생성**: 실제 뉴스/이벤트가 아닌 `|변동률| > 3%` 또는 `거래량비율 > 2` 조건으로 텍스트 자동 생성
5. **12가지 기준 채점**: 각 0/1/2점, 총 24점 만점
6. **진입 전략**: 시그널 등급에 따라 고정 %로 손절가/목표가 산출

## 2. SNS 화제 종목 탭 — 스코어링 기준 12개

| # | 기준명 | key | 실제 측정값 | 채점 기준 | 데이터 출처 |
|---|--------|-----|-----------|----------|------------|
| C1 | 가격 변동률 | priceChange | `abs(preMarketChangePercent)` | ≥3%=2, ≥1%=1 | Yahoo Finance |
| C2 | 프리마켓 거래량 | preMarketVolume | `volumeRatio` (거래량/평균×0.05) | ≥2.0=2, ≥1.0=1 | Yahoo Finance |
| C3 | 갭 유지 | gapHold | `abs((price-prevClose)/prevClose)` | ≥5%=2, ≥2%=1 | Yahoo Finance |
| C4 | 52주 최고가 근접도 | pricePattern | `(52wHigh-price)/52wHigh` | ≤5%=2, ≤10%=1 | Yahoo Finance |
| C5 | 촉매 이벤트 | newsFreshness | `catalysts.length` (자동생성 텍스트) | ≥3=2, ≥1=1 | **자동 생성** |
| C6 | 소셜 버즈 | socialBuzz | `심볼 해시 % 범위` | ≥70=2, ≥40=1 | **🔴 시뮬레이션** |
| C7 | 가격 안정성 | sectorSentiment | `(dayHigh-dayLow)/dayHigh` | ≤2%=2, ≤5%=1 | Yahoo Finance |
| C8 | 거래대금 | tradingValue | `price × volume` | ≥$50M=2, ≥$10M=1 | Yahoo Finance |
| C9 | 장중 변동폭 | intradayRange | `(dayHigh-dayLow)/prevClose` | ≥10%=2, ≥5%=1 | Yahoo Finance |
| C10 | 거래대금 효율 | tradingValueEfficiency | `abs(change%) / 거래대금(백만)` | ≥1.0=2, ≥0.3=1 | Yahoo Finance |
| C11 | 시가총액 규모 | marketCapCategory | `price × (1M + 해시 % 500M)` | 소형주=2, 중형=1 | **🔴 시뮬레이션** |
| C12 | 연속 상승일 | consecutiveUpDays | `해시 % 6 ± 변동률 보정` | ≥3일=2, ≥2일=1 | **🔴 시뮬레이션** |

**시뮬레이션 데이터 비중: C6+C11+C12 = 최대 6점 / 24점 = 25%**

## 5. 발견된 결함

### 🔴 CRITICAL — 즉시 수정 필요

#### P1: 시뮬레이션 데이터가 총점의 17~25% 차지

- **위치**: scoring.ts (C6, C11, C12), scoringUndervalued.ts (C8, C11)
- **현상**: 심볼의 문자 해시값(charCodeAt 합산)으로 결정론적 가짜 데이터 생성
- **결과**: AAPL은 시장 상황과 무관하게 **항상 같은** 소셜 버즈/시총/연속 상승일 점수를 받음
- **위험**: 사용자가 "소셜 버즈 높음", "소형주", "3일 연속 상승" 등의 표시를 보고 실제 데이터로 오인하여 투자 결정

#### P2: abs()로 인한 하락·상승 동일 취급

- **위치**: scoring.ts `scorePreMarketChange()`, `scoreGapSize()`, `scoreTradingValueEfficiency()`
- **현상**: `Math.abs(preMarketChangePercent)` 사용 → -8% 폭락 = +8% 급등 동일 점수
- **결과**: 폭락 중인 종목이 "높은 모멘텀" 시그널을 받고 상위에 랭크됨
- **위험**: 급락 종목에 대해 매수 진입 전략까지 표시

#### P3: 서로 모순되는 기준 쌍

- **위치 1**: scoring.ts C7(안정성: 스프레드↓=고점) vs C9(변동폭: 스프레드↑=고점)
  - 동일한 `dayHigh-dayLow`를 가지고 C7은 작을수록, C9는 클수록 높은 점수
- **위치 2**: scoringUndervalued.ts C2(52주저가 근접: 가까울수록=고점) vs C12(반등 강도: 멀수록=고점)
  - C2는 52주 최저가 대비 10% 이내 = 2점, C12는 52주 최저가 대비 20% 이상 이격 = 2점
  - **C2와 C12가 동시에 만점일 수 없음** → 최대 3점에서 잠김

#### P4: 기준명이 실제 측정 내용과 불일치

| 표시 이름 | 사용자가 기대하는 것 | 실제 측정 |
|-----------|---------------------|-----------|
| 섹터 심리 | 해당 섹터의 투자 심리 지수 | 개별 종목의 고가-저가 스프레드 |
| 기관 관심 거래량 | 기관 투자자의 매수/매도 신호 | 단순 거래량 비율 + 52주 저가 근접 |
| 뉴스 최신도 / 촉매 이벤트 | 실제 뉴스 피드 분석 | |변동률|>3% 또는 거래량>2x 시 자동생성 문자열 |
| 가격 추세 패턴 | 다일/주봉 추세 분석 | 프리마켓 1일치 변동률 단일값 |

### 🟠 HIGH — 잘못된 투자 판단 유발

#### P5: "저평가" 탭의 근본적 한계

- 진정한 저평가 분석에는 P/E, P/B, PEG, DCF, 부채비율 등 **펀더멘털 데이터**가 필수
- 현재는 가격/거래량 기술적 데이터만 사용 → "52주 최저가 근처" = 기업 파산 위기일 수도 있음
- Yahoo Finance 비공식 API로는 재무제표 데이터 접근 불가

#### P6: 진입 전략의 고정 % 기반 문제

- 모든 종목에 동일한 % 적용 (AAPL -2%와 페니스탁 -2%는 전혀 다른 의미)
- 실제 지지/저항선, ATR(평균 진폭), 거래량 프로파일 무시
- **HIGH 시그널 → 가장 타이트한 손절(2%)**: 모멘텀 종목의 높은 변동성에 쉽게 스탑아웃
- MODERATE 손익비 1:0.7, LOW 손익비 1:0.2 → **기대값이 음수**

#### P7: 데이터 소스 신뢰성

- Yahoo Finance 비공식 API — SLA 없음, 예고 없이 차단 가능
- 프리마켓 가격이 없으면 정규장 가격으로 폴백 → 프리마켓 분석 도구인데 정규장 데이터를 사용
- 45초 캐시 → 급변하는 프리마켓에서 stale 데이터로 의사결정

## 6. 권장 사항

1. **시뮬레이션 기준을 스코어링에서 제외하거나 가중치를 0으로 변경** — 실제 데이터 API 연동 전까지
2. **abs() 제거** — 화제 종목 탭에서는 양의 변동만 점수화하거나, 상승/하락 분리 채점
3. **모순 기준 쌍 해소** — C7/C9 중 하나 제거, 저평가 C2/C12를 하나의 복합 기준으로 통합
4. **기준명을 실제 측정값과 일치하도록 변경** — "섹터 심리" → "가격 스프레드" 등
5. **면책 조항 강화** — 시뮬레이션 데이터 비중, abs() 특성, 펀더멘털 미반영을 명시
6. **진입 전략에 경고 추가** — 고정 % 기반의 한계, 실제 트레이딩 전 전문가 상담 권장
